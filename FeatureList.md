## 2. 产品原则

1. **声音优先**：核心目标是建立声音与字符之间的直接反射，不能只训练对点划文本的视觉翻译。
2. **科学训练**：采用渐进学习、Farnsworth 间隔、间隔重复和错题强化等方法。
3. **本地优先**：无账户、无网络时仍可完成核心课程，并在本地保存进度。
4. **低延迟反馈**：声音、触摸、键盘和视觉反馈必须及时、稳定且可校准。
5. **跨端一致**：训练规则、评分方式和数据结构在各平台保持一致，平台差异仅限系统能力。
6. **渐进增强**：先完成可靠的学习闭环，再增加云同步、系统输入法和社区功能。
7. **隐私优先**：输入内容和训练记录默认保存在本地；涉及输入法时不得记录敏感输入。

## 4. 用户与核心使用场景

### 4.1 目标用户

- 第一次接触 Morse Code 的入门用户。
- 希望提高听抄和发报速度的进阶用户。
- 无线电爱好者、应急通信学习者及相关课程学员。
- 希望使用 Morse Code 进行替代输入或趣味输入的用户。

### 4.2 核心训练闭环

1. 学习字符及对应声音。
2. 识别字符、点划文本或声音。
3. 使用按键发出目标字符、单词或句子。
4. 获得即时反馈和错误解释。
5. 系统记录错误、反应时间和节奏偏差。
6. 后续课程自动强化薄弱内容。

## 5. P0：MVP 功能

### 5.1 首次使用与导航

- [x] 提供简短的新手引导，说明点、划、间隔和输入方式。
- [x] 用户可跳过引导并稍后重新查看。
- [x] 提供首页、学习、练习、自由发报、统计和设置入口。
- [x] 自动恢复用户上次未完成的课程或练习设置。
- [x] 不注册账户即可使用全部 P0 功能。

**验收标准**

- 新用户能够在不阅读外部说明的情况下开始第一次字符练习。
- PC 和移动端均能在三次操作内进入最近一次训练。

### 5.2 Morse Code 标准与字符库

- [x] 以 ITU-R M.1677 国际摩尔斯电码为基础维护字符映射。
- [x] 支持 A-Z 英文字母。
- [x] 支持 0-9 数字。
- [x] 支持常用标点，并明确展示暂未支持的字符。
- [x] 遵循标准时间比例：点 1 单位、划 3 单位、元素间隔 1 单位、字符间隔 3 单位、单词间隔 7 单位。
- [x] 编码、解码、播放和评分共用同一份字符数据源。
- [x] 对大小写、连续空格和非法输入进行规范化处理。

**验收标准**

- 字符库中的每个受支持字符均通过编码、解码和时序单元测试。
- 无法识别的输入不会导致崩溃，且会给出明确提示。

### 5.3 基础学习

- [ ] 字符卡片展示字符、点划表示和播放按钮。
- [x] 支持逐个字符学习。
- [x] 支持按字母、数字、标点筛选。
- [x] 支持重复播放当前字符。
- [x] 学习字符时支持按当前键位直接发报，并提供声音与即时正误反馈。
- [ ] 展示当前字符在标准节奏下的声音和动画。
- [ ] 允许收藏或标记需要加强的字符。

### 5.4 基础练习模式

- [x] 字符 → 用户选择或输入 Morse Code。
- [x] Morse Code 文本 → 用户输入字符。
- [x] Morse Code 声音 → 用户输入字符。
- [x] 字符 → 用户通过真实按键时长发出 Morse Code。
- [x] 支持设置字符范围、题目数量和是否打乱顺序。
- [x] 每题提供正确、错误和超时反馈。
- [x] 错误后可播放或演示正确答案。
- [x] 完成后展示正确率、用时、平均反应时间和错题列表。
- [x] 支持只重练本轮错题。

**验收标准**

- 四种基础练习模式使用统一的题目与评分模型。
- 用户刷新页面或短暂切换应用后，已完成的题目记录不会丢失。

### 5.5 自由发报与输入

- [x] 支持单键模式：根据按下时长识别点或划。
- [x] 支持双键模式：分别输入点和划。
- [x] 支持键盘、鼠标和触摸屏。
- [x] 输入过程中实时播放对应音调。
- [x] 根据停顿自动判断元素、字符和单词边界。
- [x] 自由发报的字符自动提交等待可调，默认使用适合连续点划输入的 6 个时间单位。
- [x] 实时显示已识别的点划和文本。
- [x] 支持退格、清空、空格和重新播放。
- [x] 在学习和发报练习中实时显示当前按下时长、点划阈值和判定结果；毫秒读数、填充条和指针必须在松开前随动画帧连续刷新。
- [x] 学习页在输入停顿达到阈值后自动判定正误、展示结果并清空，支持连续重复练习且无需退格/清空按钮。
- [x] 单键和双键练习均实时显示真实物理按压时长；双键的点划类型仍由按键本身决定。
- [ ] 展示输入相对标准节奏的偏差分布。

**验收标准**

- 按键按下时开始发声，释放时停止发声，无明显可感知卡顿。
- 用户可调整点划判定阈值，并能恢复默认值。

### 5.6 音频系统

- [ ] 使用音频时间轴调度 Morse Code，不使用普通定时器作为最终播放时钟。
- [ ] 支持播放、暂停、停止、重复和单题重播。
- [x] 支持调节音量。
- [x] 支持调节音调频率。
- [x] 支持至少正弦波和方波两种音色。
- [x] 支持独立设置字符速度（Character WPM）和有效速度（Effective WPM）。
- [x] 使用平滑音量包络，避免声音首尾爆音。
- [ ] 正确、错误、完成等界面音效可独立关闭。
- [ ] 页面首次获得用户交互后正确初始化音频。
- [ ] 应用从后台恢复后可自动修复失效的音频上下文。

**验收标准**

- 标准点划和间隔比例通过自动化时序测试。
- 连续播放长句时不会因界面动画或渲染负载产生累计节奏漂移。

### 5.7 输入与自定义按键

- [x] 用户可自定义点、划、提交、删除、重播和暂停按键。
- [x] 检测并提示应用内按键冲突。

## 10. 非功能需求

### 10.1 性能

- 首次缓存完成后，核心页面应能快速离线启动。
- 输入反馈必须优先于非关键动画和统计计算。
- 长句播放时不得出现随播放长度增加的累计时序漂移。
- 低性能移动设备上仍应保持按键、声音和视觉反馈同步。

### 10.2 可靠性

- 训练过程中的单题结果应及时持久化。
- 数据迁移失败时保留原始数据并提供恢复路径。
- 音频、振动或云端服务不可用时，核心训练仍可降级运行。

### 10.3 安全与隐私

- 默认仅在本地保存数据。
- 明确告知用户哪些数据会被同步。
- 不收集应用内输入法和系统输入法中的正文内容用于分析。
- 云端数据使用传输加密，并提供删除能力。
- 日志不得包含密码、令牌、完整输入正文或其他敏感数据。

### 10.4 兼容性

- Web 端优先支持当前主流 Chromium、Safari 和 Firefox。
- 每次发布至少验证一台触摸设备和一种桌面键盘环境。
- 原生包装版本必须与同版本 Web 核心使用相同字符表和训练规则。

### 10.5 可测试性

- Morse 编解码、时间比例、评分和复习算法必须可脱离界面测试。
- 音频调度与界面渲染逻辑分离。
- 所有 P0 用户主流程应具备端到端测试。
- 平台特有功能应具有设备测试清单。

## 11. MVP 发布判定

同时满足以下条件才可将版本标记为 MVP：

- [ ] 所有 P0 功能完成，或存在经记录批准的范围调整。
- [ ] 四种基础训练模式可以完整开始、作答、反馈和结算。
- [ ] Morse 编解码和标准时序自动化测试全部通过。
- [ ] Web、Android/iOS 目标浏览器环境及桌面端的核心音频交互验证通过。
- [ ] 离线启动、训练和数据恢复验证通过。
- [ ] PC、手机和平板关键页面无阻塞性布局问题。
- [ ] 键盘和触摸输入不存在已知的阻塞性延迟问题。
- [ ] 无导致学习记录丢失的已知严重缺陷。
- [ ] 隐私说明、数据清空和导出入口可用。

## 12. 当前明确不进入 MVP 的范围

- 系统级输入法。
- 用户社区、聊天和公开排行榜。
- 付费订阅和广告系统。
- 教师后台和班级管理。
- 外接硬件电键的完整支持。
- 复杂的无线电信道模拟。
- 必须联网才能使用的功能。

## 13. 功能变更流程

1. 新功能先描述用户问题和预期价值，不直接以技术方案代替需求。
2. 标注优先级、目标平台、依赖项和验收标准。
3. 评估是否影响 Morse 核心、音频时序、数据格式或平台权限。
4. 影响 MVP 范围的变更必须记录决定、理由和日期。
5. 开发任务、设计稿和测试用例应引用本文档中的对应章节。
6. 功能完成后只有通过验收标准才能勾选，不以“代码已提交”作为完成定义。
