## 2. 产品摘要

Learning Morse Code 是一个声音优先、离线可用、可跨平台发布的 Morse Code 学习和输入工具。它帮助用户完成从“认识点划”到“能够听抄和稳定发报”的训练闭环，同时保持足够简洁的日常练习体验。

产品不以转换器或字符表为终点。字符表和文本转换只是辅助能力，核心价值是：

- 建立声音与字符之间的直接反射。
- 通过即时反馈纠正识别和发报节奏。
- 根据历史表现持续强化薄弱字符。
- 让同一套学习进度和操作习惯自然延伸到 Web、移动端和桌面端。

## 3. 用户问题

### 3.1 核心问题

1. 许多入门工具强调视觉点划，用户会“看码”，却不能直接听出字符。
2. 字符表、播放器、听抄、发报和统计通常分散在不同工具中，训练无法形成闭环。
3. 普通网页训练器的按键声音与节奏容易受渲染和定时器影响，发报手感不稳定。
4. 用户不知道自己容易混淆哪些字符，也不知道下一次应该练什么。
5. 训练常发生在网络不稳定或不方便登录的环境，强制账户会提高使用门槛。

### 3.2 机会

用一个低延迟的音频与输入核心统一“听、认、敲、译”四类训练，并以本地优先方式保存进度，可以形成比简单转换器更持久的产品价值。后续的应用内 Morse 输入法和系统级输入法可以复用同一输入引擎，成为产品差异化能力。

## 4. 产品目标与非目标

### 4.1 MVP 目标

- 新用户无需外部教程即可完成第一次声音识别练习。
- 用户能够完成字符学习、四种基础练习、自由发报和结果复盘。
- 核心训练在断网、无账户状态下完整可用。
- 声音播放和按键反馈在目标设备上稳定，无明显累计漂移或可感知卡顿。
- 用户能够查看薄弱字符并立即开始针对性复习。
- PC 与移动端共享一致的概念、术语、评分方式和数据格式。

### 4.2 MVP 非目标

- 不提供系统级输入法。
- 不提供公开社区、聊天、排行榜或课程市场。
- 不提供付费、广告或复杂商业化系统。
- 不模拟完整无线电信道或 QSO 流程。
- 不要求登录，也不在 MVP 中实现云同步。
- 不承诺第一版同时上架全部应用商店。

## 5. 目标用户与任务

### 5.1 入门学习者

**特征**：不了解时间比例，容易把 Morse 当作需要背诵的视觉符号。

**核心任务**：

- 当我第一次学习 Morse Code 时，我希望快速理解声音和节奏，以便不依赖死记点划字符串。
- 当我完成一小段练习时，我希望知道哪些字符掌握不好，以便立即复习。

### 5.2 进阶训练者

**特征**：认识大部分字符，但听抄速度、反应速度或发报节奏不稳定。

**核心任务**：

- 当我进行日常练习时，我希望快速选择字符范围和速度，以便针对薄弱项训练。
- 当我发报时，我希望看到点、划和间隔偏差，以便改善节奏稳定性。

### 5.3 无线电相关用户

**特征**：后续需要呼号、数字组、Q 简语、RST 和 QSO 训练。

**MVP 关系**：MVP 先提供可靠的字母、数字、常用标点和自定义字符范围；专业内容进入 P1/P2。

### 5.4 趣味或辅助输入用户

**特征**：对单键输入、双键输入或替代输入方式感兴趣。

**MVP 关系**：通过自由发报验证输入引擎；应用内输入法进入 P1，系统级输入法进入 P2/P3。

## 6. MVP 产品假设与成功判定

### 6.1 待验证假设

| 编号 | 假设 | MVP 验证方式 |
| --- | --- | --- |
| H-01 | 声音优先的新手引导可以让用户独立完成首轮练习 | 可用性测试中首次任务完成率 |
| H-02 | 四种基础模式足以形成初步“听、认、敲、译”闭环 | 用户能从结果页进入错题重练并完成第二轮 |
| H-03 | 本地优先可降低首次使用门槛 | 首次启动不要求登录，离线主流程测试通过 |
| H-04 | 实时按键音和节奏反馈是自由发报的核心价值 | 设备测试中无明显卡顿；测试用户能理解偏差反馈 |
| H-05 | 薄弱字符统计能推动下一次训练 | 用户能从首页或统计页一键进入薄弱项练习 |

### 6.2 MVP 成功指标

MVP 阶段不默认上传个人训练明细。以下指标优先通过本地汇总、受控测试和用户主动反馈评估：

- 首次使用者完成引导后，能够独立完成一轮 10 题声音识别练习。
- 核心可用性测试中，至少 80% 的测试用户无需帮助完成“开始练习—作答—查看结果—重练错题”流程。
- Morse 编解码及标准时序自动化测试通过率为 100%。
- 目标设备上连续播放测试不出现随句长增长的累计时序漂移。
- 已完成首次加载的设备在断网后仍能进入四种基础练习和自由发报。
- 训练过程中刷新或切换前后台后，不丢失已提交的题目记录。
- 严重可访问性、数据丢失和核心输入阻塞问题为 0。

## 7. 当前 MVP 决策

以下是进入交互设计和技术验证所需的当前基线。若验证失败，按 `FeatureList.md` 的功能变更流程调整。

| 决策项 | 当前决定 | 理由 |
| --- | --- | --- |
| 首发顺序 | PWA 优先；Web 核心通过后再验证 Capacitor/Tauri 包装 | 优先验证音频、输入和学习闭环 |
| 账户 | MVP 不提供账户 | 降低门槛并落实本地优先 |
| 界面语言 | 简体中文和英文；首次跟随系统，可手动切换 | 服务初期目标用户并为国际内容预留 |
| 字符范围 | A-Z、0-9、`.` `,` `?` `/` `=` `+` `-` `(` `)` `:` `'` | 覆盖基础学习和常见训练文本 |
| 默认主题 | 跟随系统；首次无法判断时使用深色 | 兼顾跨设备体验 |
| 默认音频 | 600 Hz、正弦波、60% 音量 | 舒适、中性的入门起点，可随时调整 |
| 默认速度 | Character WPM 20、Effective WPM 10 | 保持字符声音整体性，同时给予入门者作答时间 |
| 默认练习长度 | 首次引导 10 题；普通练习 20 题 | 首次体验短，常规训练仍可形成统计意义 |
| 练习超时 | 默认关闭；用户可开启 | 避免入门阶段制造不必要压力 |
| 单键判定 | 以当前点单位为基础，2 个单位为点/划初始阈值，可校准 | 位于标准点和划时长之间，便于解释 |
| 自动提交 | 元素停顿 1 单位、字符停顿 3 单位、单词停顿 7 单位 | 遵循标准时间比例 |
| 统计存储 | 本地保存逐题结果和会话摘要；正文型自定义输入不写入日志 | 支持错题分析并保护输入隐私 |
| Koch 课程 | P1 实现；顺序和解锁阈值在 P1 规格中确定 | 不阻塞 P0 学习闭环验证 |

## 8. 核心用户流程

### 8.1 首次使用

1. 用户选择或确认界面语言。
2. 产品用一个简短声音示例解释点、划和间隔。
3. 用户点击或按键，产品由此解锁浏览器音频能力。
4. 用户选择主要输入方式：键盘、触摸或鼠标；可跳过校准。
5. 用户完成 10 题引导练习。
6. 产品展示简短结果并推荐“继续学习”或“自由练习”。
7. 引导完成状态保存在本地，之后进入首页。

### 8.2 学习一个字符

1. 用户从“学习”进入字符表。
2. 选择字符后查看字符、点划表示和节奏动画。
3. 播放声音，可重复播放。
4. 将字符标记为需要加强，或直接开始包含该字符的练习。

### 8.3 完成一次基础练习

1. 用户从首页快捷入口或练习中心选择模式。
2. 使用快速开始，或修改字符范围、题量、速度和随机顺序。
3. 产品逐题显示或播放题目。
4. 用户作答并得到即时反馈。
5. 每个已提交答案立即写入本地会话。
6. 完成后显示结果摘要、薄弱字符和错题。
7. 用户可以重练错题、再来一组或返回练习中心。

### 8.4 自由发报

1. 用户选择单键或双键模式。
2. 页面展示当前键位、速度和点划判定阈值。
3. 用户按下时立即发声，释放时停止。
4. 系统显示点划组合，并根据停顿生成字符和单词。
5. 用户可删除、清空、复制或重新播放内容。
6. 本模式默认不将完整输入正文写入训练统计。

### 8.5 薄弱项复习

1. 用户从首页或统计页看到薄弱字符。
2. 点击“强化练习”。
3. 产品预填包含薄弱字符的练习设置。
4. 用户确认后直接进入练习。
5. 新结果更新字符能力统计。

## 9. 功能行为规格

### 9.1 新手引导

- **PS-ONB-001**：首次启动显示可跳过的引导，之后可从设置重新打开。
- **PS-ONB-002**：引导必须包含一次由用户主动触发的声音播放，以初始化音频上下文。
- **PS-ONB-003**：引导不得要求注册、授权通知或提供个人信息。
- **PS-ONB-004**：跳过引导的用户应进入首页，并看到“开始第一次练习”的明确入口。
- **PS-ONB-005**：引导中断时保存当前步骤；重新进入可继续或重新开始。

### 9.2 首页

- **PS-HOME-001**：返回用户优先看到“继续上次训练”或“开始推荐练习”。
- **PS-HOME-002**：新用户优先看到第一次学习入口，不展示空白统计图。
- **PS-HOME-003**：首页提供四种基础模式和自由发报的快捷入口，但避免同时展示全部高级设置。
- **PS-HOME-004**：当存在足够训练数据时，首页展示最多 5 个薄弱字符及强化入口。
- **PS-HOME-005**：离线状态不得遮挡核心操作，只以非阻塞状态提示表示。

### 9.3 字符学习

- **PS-LEARN-001**：字符表按字母、数字和标点分组，并允许搜索。
- **PS-LEARN-002**：字符详情至少包括字符、点划、声音播放、节奏动画和加强标记。
- **PS-LEARN-003**：用户切换字符时应停止旧字符尚未完成的演示，避免声音重叠。
- **PS-LEARN-004**：播放不得自动循环；循环必须由用户明确开启。
- **PS-LEARN-005**：从字符详情开始练习时，目标字符必须包含在预填字符集中。

### 9.4 练习配置

- **PS-SETUP-001**：快速开始使用上一次该模式的有效设置；首次使用采用第 7 节默认值。
- **PS-SETUP-002**：自定义设置至少包括字符范围、题量、Character WPM、Effective WPM 和随机顺序。
- **PS-SETUP-003**：声音识别模式不得关闭声音；其他模式可按规则显示适用设置。
- **PS-SETUP-004**：无有效字符时禁止开始，并解释如何修复。
- **PS-SETUP-005**：修改练习设置不会改变全局默认，除非用户明确选择“保存为默认”。

### 9.5 练习会话

- **PS-SES-001**：四种模式使用统一会话状态：准备、出题、作答、反馈、完成、暂停和中断。
- **PS-SES-002**：页面每次只突出一个主要任务，避免结果统计在作答期间分散注意力。
- **PS-SES-003**：提交后锁定当前答案，记录结果，再进入反馈状态。
- **PS-SES-004**：错误反馈必须同时提供正确字符和可播放的正确声音，不只用颜色表示。
- **PS-SES-005**：声音题允许重播；重播次数写入题目结果，但 MVP 不因此扣分。
- **PS-SES-006**：暂停时停止尚未开始的音频调度，恢复后重新播放当前题，不从中间位置继续。
- **PS-SES-007**：离开未完成会话时提示“继续保存”或“结束并结算”；应用异常关闭时默认保留会话。
- **PS-SES-008**：题目已提交后立即持久化，不能等到整组练习结束。

### 9.6 评分规则

- **PS-SCORE-001**：字符识别题采用规范化后的精确匹配，英文字母不区分大小写。
- **PS-SCORE-002**：点划文本忽略首尾空格，接受 `.`/`·` 和 `-`/`—` 的规范化输入。
- **PS-SCORE-003**：发报题分别记录字符是否正确和节奏偏差；MVP 总正确率以字符正确为主。
- **PS-SCORE-004**：反应时间从题目可作答时开始，到首次有效提交时结束；暂停时间不计入。
- **PS-SCORE-005**：未启用超时时，不产生超时错误；启用后超时题进入错题列表。
- **PS-SCORE-006**：错误后修改答案不改变该题首次结果，但可记录“纠正后正确”。

### 9.7 自由发报

- **PS-KEY-001**：按键按下触发发声，释放停止；自动重复键盘事件不得重复开始声音。
- **PS-KEY-002**：浏览器窗口失去焦点或触摸被取消时，必须强制结束当前音符，防止持续发声。
- **PS-KEY-003**：单键模式根据按下时长和阈值判定点划；双键模式不使用时长判断符号类型。
- **PS-KEY-004**：自动提交字符后仍允许用户执行撤销或退格。
- **PS-KEY-005**：无法解码的组合保持可见并标记为未知，不静默丢弃。
- **PS-KEY-006**：复制正文必须由用户明确触发；默认不保存完整自由输入正文。

### 9.8 结果和统计

- **PS-STAT-001**：结果页优先显示正确率、用时、平均反应时间和错题数。
- **PS-STAT-002**：当没有错题时，主操作为“提高难度”或“再来一组”；存在错题时主操作为“重练错题”。
- **PS-STAT-003**：薄弱字符至少综合近期错误率和作答次数，样本不足时标记为“数据不足”。
- **PS-STAT-004**：删除训练历史后重新计算全部聚合统计。
- **PS-STAT-005**：统计页无数据时展示可行动的练习入口，不展示误导性的零值趋势。

### 9.9 设置、数据与隐私

- **PS-SET-001**：设置分为外观、音频、输入、训练、数据和关于六组。
- **PS-SET-002**：高风险操作（清空数据、覆盖导入）必须二次确认并说明影响。
- **PS-SET-003**：导出文件包含格式版本、导出时间、设置、进度和训练记录，不包含密钥或设备标识。
- **PS-SET-004**：导入前验证版本和结构；失败时不得破坏现有数据。
- **PS-SET-005**：按键冲突在保存前明确标记；系统保留组合键显示警告。
- **PS-SET-006**：隐私说明使用简明语言说明本地保存、未来同步和自由输入正文策略。

## 10. 训练与音频业务规则

### 10.1 标准时序

- 点：1 个时间单位。
- 划：3 个时间单位。
- 同一字符内元素间隔：1 个时间单位。
- 字符间隔：3 个时间单位。
- 单词间隔：7 个时间单位。
- 基础点单位由 Character WPM 决定；Farnsworth 仅扩展字符和单词之间的间隔，不拉长字符内部点划。

### 10.2 速度约束

- Character WPM 不得低于 Effective WPM。
- MVP 允许范围建议为 5–60 Character WPM、3–60 Effective WPM，最终边界由音频实验验证。
- 速度调整实时更新后续播放，不修改已生成并开始播放的当前音频片段。

### 10.3 题目生成

- 同一组练习避免连续出现超过 3 次完全相同的目标。
- 随机模式必须能通过种子重现，以支持问题排查和测试。
- 错题重练至少包含本轮错题；题量不足时可补充相同字符的新题。
- 生成后应保存本次题目快照，字符库或设置更新不得改变进行中的会话。

### 10.4 本地能力数据

每个字符至少维护：

- 总作答次数和近期作答次数。
- 正确次数和错误次数。
- 平均反应时间。
- 最近练习时间。
- 练习模式维度的表现。
- “需要加强”手动标记。

薄弱项算法在 MVP 可采用透明、可测试的加权分数；不得使用无法解释且缺少样本保护的黑盒排名。

## 11. 状态与异常处理

| 状态 | 产品行为 |
| --- | --- |
| 首次音频尚未解锁 | 显示“点击启用声音”，不自动播放失败 |
| 音频初始化失败 | 提供重试、静音继续和诊断提示 |
| 浏览器/应用进入后台 | 结束实时输入音符，暂停会话或安全恢复 |
| 离线 | 保持核心功能可用，显示非阻塞离线标记 |
| 本地存储不可用 | 明确提示进度无法保存，允许临时体验 |
| 数据导入失败 | 保留原数据，说明失败原因 |
| 不支持字符 | 在开始训练或播放前集中提示 |
| 按键冲突 | 阻止保存冲突配置，或要求用户明确确认可覆盖冲突 |
| 会话异常中断 | 下次启动提供继续或结束会话 |
| 无统计数据 | 展示练习入口和数据如何产生的说明 |

## 12. 内容与文案原则

- 使用“点”“划”“字符速度”“有效速度”等一致术语。
- 首次出现 WPM 时提供简短解释。
- 正确反馈简短明确；错误反馈说明正确答案，不使用羞辱性文案。
- 高级概念采用渐进披露，默认界面不要求入门用户理解全部参数。
- 中英文字符内容共用相同 Morse 数据；界面文案和课程说明独立本地化。
- 声音不能作为唯一的信息载体，视觉状态也不能只依赖颜色。

## 13. 平台和发布策略

### 13.1 MVP 发布阶段

1. 在现代桌面和移动浏览器验证 PWA。
2. 完成离线安装、音频恢复和触摸输入测试。
3. 冻结 Web 核心的第一版数据结构和训练接口。
4. 使用 Capacitor 验证 Android/iOS 容器。
5. 使用 Tauri 验证 Windows/macOS/Linux 桌面包装。

原生容器验证失败不会阻塞 PWA MVP，但必须记录平台差异，且不能通过改变训练规则形成分叉版本。

### 13.2 最低测试环境

- 一种 Chromium 桌面浏览器。
- Safari 桌面或移动浏览器。
- Firefox 桌面浏览器。
- 至少一台 Android 触摸设备。
- 至少一台 iPhone 或 iPad。
- 一种 Windows 或 macOS 物理键盘环境。

具体版本范围在工程初始化时由 `Architecture.md` 和发布支持策略确定。

## 14. 风险与验证计划

| 风险 | 影响 | 首个验证动作 | 退出条件 |
| --- | --- | --- | --- |
| Web Audio 调度漂移 | 声音训练失真 | 构建长句调度实验 | 长句无累计漂移，前后台可恢复 |
| 键盘/触摸输入延迟 | 发报手感不可用 | 构建实时按键实验 | 目标设备无明显卡顿或粘连音 |
| 单键阈值不适合不同用户 | 点划误判 | 提供阈值和校准实验 | 用户可在短校准后稳定输入 |
| 移动端音频策略差异 | 首次播放或恢复失败 | 真机测试音频解锁/恢复 | 有明确解锁、重试和降级路径 |
| 离线更新损坏本地数据 | 学习进度丢失 | 数据版本和升级测试 | 更新失败可保留旧数据并恢复 |
| 首页信息过多 | 入门用户迷失 | 线框可用性测试 | 首次任务三次操作内可达 |

## 15. MVP 验收场景

### 场景 A：新用户完成声音练习

给定一台从未使用过本产品的移动设备，当用户打开应用并跟随引导时，用户能够解锁声音、听到题目、完成 10 题、查看结果，并进入下一项练习；全程不要求注册。

### 场景 B：返回用户重练薄弱字符

给定已有至少两次练习数据的用户，当其打开首页时，可以看到薄弱字符并在三次操作内开始预填好的强化练习；完成后统计更新。

### 场景 C：断网训练

给定已完成首次加载的设备，当网络断开后重新打开应用，用户仍可进入字符学习、四种基础练习、自由发报和统计页面，新的记录在本地保存。

### 场景 D：自由发报安全恢复

给定用户正在按住发报键，当窗口失焦、触摸取消或应用进入后台时，声音立即结束；返回应用后不会持续发声，用户可继续输入。

### 场景 E：数据备份恢复

给定用户导出的有效备份，当其在全新数据环境中导入时，设置、训练记录和进度被恢复；无效备份不得覆盖现有数据。
