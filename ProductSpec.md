## 2. V2 产品摘要

Learning Morse Code 是一个声音优先、离线可用、可跨平台发布的专业 CW 学习和训练工具。它覆盖从基础认知到连续听抄、稳定发报以及专业查询转换的完整路径，同时让入门用户只看到当前需要的难度。

V2 由四大功能域构成：

1. **基础学习与识别**：从不会到建立字符、声音与节奏的直接联系。
2. **听抄与接收训练**：从单字符听辨到连续报文和专业通信内容接收。
3. **发报与节奏训练**：从目标跟发到自由拍发和节奏稳定性分析。
4. **查询与转换工具**：完成 Morse、专业术语和中文电码的查询、转换、播放与导出。

首页、进度与统计、设置为全局支持页面。转换器和字符表不是产品终点；产品核心价值仍然是训练闭环、低延迟反馈和可解释的能力成长。

## 3. 产品问题与机会

### 3.1 用户问题

1. 只背视觉点划的用户往往无法直接听出字符。
2. 单字符学习、连续抄报、发报和专业内容常分散在多个工具中，设置和记录不能延续。
3. “字母练习、字母抄报、随机字母”按页面堆叠后功能重复，用户也难以理解差别。
4. 普通网页发报反馈延迟或节奏漂移，会训练错误的动作记忆。
5. 无线电用户需要呼号、RST、Q 简语、程序信号和数字简码，而入门工具通常缺少专业进阶路径。
6. 汉字与 Morse 的转换经常忽略中文电码这一中间层，容易产生概念错误或不可追溯结果。

### 3.2 产品机会

将“训练方式”和“训练内容”拆开，并以统一会话引擎组合，可以用较少的页面覆盖更多专业场景：同一份词组内容既能用于视觉识别、听抄，也能用于目标跟发；同一份 Q 简语数据既能查询，也能进入收报和发报训练。

## 4. 产品目标与非目标

### 4.1 V2 结构重构目标

- 用户能够清楚区分学习、听抄、发报和工具四类目标。
- 旧四种基础练习迁入新功能域后，历史会话、错题和统计保持可用。
- 训练内容只维护一份，通过预设组合到不同训练方式。
- 首页之外的一级功能入口与四大功能域一一对应。
- 现有实时音频、按键输入、按压时长、自动判定和离线能力无回归。
- 新结构允许逐步加入词组、长文、呼号、Q 简语和数字简码，而不再复制训练页面。

### 4.2 V2 正式版目标

- 新用户无需外部教程即可完成第一轮声音训练。
- 进阶用户可以完成随机字符组、词组、短句、长文和自定义文本听抄。
- 用户可以完成目标跟发、自由拍发和可解释的节奏复盘。
- 无线电用户可以训练呼号、RST、Q 简语、CW 缩略语、程序信号和数字简码。
- 用户可以完成 Morse 与文本、汉字与中文电码的可靠查询和转换。
- 核心训练在断网、无账户状态下完整可用。

### 4.3 本次结构重构非目标

- 不在一次重构中实现全部 P1/P2 内容。
- 不提供完整 QSO 仿真、复杂信道模拟或比赛日志系统。
- 不提供系统级输入法、账户同步、社区、排行榜或商业化系统。
- 不在中文电码数据源、版本和许可未确认前发布生产转换结果。
- 不为了新页面结构重写已经稳定的音频、输入、会话和存储核心。

## 5. 目标用户与核心任务

### 5.1 入门学习者

- 我想按循序渐进的课程学习字符声音，而不是死记点划图形。
- 我想在学习字符时马上听、看、敲，并知道哪里出错。
- 我想让系统告诉我下一组应该学什么。

### 5.2 接收能力训练者

- 我想选择字母、数字、词组或自定义内容进行听抄。
- 我想分别设置字符速度和有效速度，并逐渐减少间隔。
- 我想进行不中断的长文抄收，并在结束后统一对照。

### 5.3 发报能力训练者

- 我想按目标字符、词组或报文跟发，并获得内容和节奏两类反馈。
- 我想自由拍发，实时听到按键声音并看到实际按压和间隔。
- 我想了解点、划和间隔是否稳定，而不只知道最终字符对不对。

### 5.4 无线电相关用户

- 我想专项训练呼号、RST、Q 简语、CW 缩略语、程序信号和数字简码。
- 我想从查询页直接进入对应听抄或跟发训练。
- 我想逐步适应噪声、衰落和不规则真实信号。

### 5.5 查询与转换用户

- 我想快速完成文本和 Morse 的双向转换、播放和复制。
- 我想查询专业缩略语的含义和发送方式。
- 我想把汉字转换成可追溯的四位中文电码，再按需要播放为数字 Morse。

## 6. 产品模型

### 6.1 三个可组合对象

| 对象 | 定义 | 示例 |
| --- | --- | --- |
| 功能域 | 用户本次要提升或完成的目标 | 听抄与接收 |
| 训练预设 | 题面、作答、反馈和默认设置的组合 | 随机字符组听抄 |
| 内容包 | 可复用、带版本的训练数据 | 英文字母、Q 简语、常用词组 |

训练会话由“功能域 + 训练预设 + 内容包 + 用户设置”生成。页面名称变化不得改变已保存会话的稳定标识。

### 6.2 计分训练与工具操作

- 计分训练进入统一会话，记录题目、首次答案、正确率、反应时间、重播和适用的节奏数据。
- 查询、转换、自由正文输入默认不计分，不把完整正文写入训练历史。
- 用户从工具进入“训练这个内容”时，系统创建新的训练会话，只保存必要的目标标识或经用户确认的内容快照。

### 6.3 内容复用规则

- 字母、数字、标点、词组、Q 简语等数据不按页面复制。
- 内容包必须包含稳定 ID、显示名称、版本、语言/地区、字符合法性和来源信息。
- 训练开始时保存内容快照或内容版本；后续内容更新不得改变进行中的题目。
- 专业内容的查询解释和训练答案来自同一数据源。

## 7. 关键产品决策

| 决策项 | V2 决定 | 理由 |
| --- | --- | --- |
| 一级功能域 | 基础学习、听抄接收、发报节奏、查询转换 | 与用户目标对齐，不按技术模式堆页面 |
| 首页 | 保留为启动和推荐入口，不算第五功能域 | 承载继续训练和跨域推荐 |
| 统计 | 重命名为“进度”，作为全局页面 | 统计服务所有功能域，不应与训练域并列 |
| 旧四种练习 | 迁移为新功能域下的训练预设 | 保留实现并减少重复概念 |
| 听抄练习器 | 定义为统一训练台 | 它是配置与会话外壳，不是内容类型 |
| 词组内容 | 一份内容，多域复用 | 避免词组练习和词组抄报重复维护 |
| 专业术语 | Q 简语、CW 缩略语、程序信号、数字简码分别建模 | 避免 CQ/QC/Q 信号混淆 |
| 中文电码 | 汉字 ↔ 四位代码 ↔ 数字 Morse 分层 | 不把国际 Morse 描述成直接支持汉字 |
| 首发顺序 | PWA 优先，再验证 Capacitor/Tauri | 延续现有跨端路线 |
| 数据策略 | 本地保存训练；自由正文与转换正文默认不记日志 | 支持复盘并保护隐私 |

## 8. 核心用户流程

### 8.1 新手完成第一次课程

1. 用户从首页进入“基础学习”。
2. 产品解释点、划、声音和间隔并解锁音频。
3. 用户完成当前双字符或 Koch 课程步骤。
4. 课程包含试听、识别和跟敲，不要求用户切换功能域。
5. 产品展示结果并推荐下一课或薄弱字符复习。

### 8.2 完成一次听抄

1. 用户进入“听抄”，选择随机字符组、词组、长文或专业预设。
2. 快速开始沿用该预设最近设置；也可进入听抄训练台调整参数。
3. 产品播放逐题或连续内容，用户键入答案或选择纸笔模式。
4. 逐题模式即时反馈；连续模式结束后统一对照。
5. 结果区分正确率、反应时间、重播次数和内容类别表现。

### 8.3 完成一次目标跟发

1. 用户进入“发报”，选择字符、词组、专业内容或自定义目标。
2. 产品按设置显示、隐藏或先播放目标。
3. 用户使用单键或双键拍发；按压期间实时显示时长和点/划状态。
4. 系统分别判断内容正确性和适用的节奏质量。
5. 达到停顿阈值后自动判定、显示结果并准备下一题。

### 8.4 自由拍发

1. 用户进入“发报”并选择“自由拍发”。
2. 用户选择单键或双键，确认当前键位和阈值。
3. 按下立即发声，释放停止；页面实时显示按压、组合和解码文本。
4. 用户可以编辑、复制或播放结果。
5. 完整正文默认不进入训练历史。

### 8.5 查询并发起训练

1. 用户在“工具”查询字符、Q 简语、缩略语、程序信号或数字简码。
2. 详情展示含义、Morse、声音和适用说明。
3. 用户选择“听抄这个内容”或“跟发这个内容”。
4. 系统带着内容包 ID 和目标项进入对应训练设置或快速开始。

### 8.6 中文电码转换

1. 用户选择汉字转中文电码或中文电码转汉字。
2. 系统显示当前码表来源和版本。
3. 未收录字、异体字和一字多码显示明确状态，用户确认候选。
4. 用户可复制四位数字组，或继续转换为数字 Morse 并播放。
5. 转换正文默认不保存。

## 9. 功能行为规格

### 9.1 首页与全局入口

- **PS2-HOME-001**：首页优先展示继续会话或推荐的下一项训练。
- **PS2-HOME-002**：首页清晰展示四大功能域入口，但不同时展开全部子预设。
- **PS2-HOME-003**：新用户看到“开始基础课程”，不展示无意义的空统计。
- **PS2-HOME-004**：进度和设置保持全局可达，但不占据四大功能域的语义位置。
- **PS2-HOME-005**：未实现的功能不显示可点击空卡片；可以在非主操作位置展示路线说明。

### 9.2 基础学习与识别

- **PS2-LEARN-001**：新手课程、字符学习和基础识别拥有清晰分区。
- **PS2-LEARN-002**：字符详情至少包含字符、点划、声音、节奏提示和跟敲入口。
- **PS2-LEARN-003**：课程可以组合试听、识别和跟敲，但课程进度只归属基础学习域。
- **PS2-LEARN-004**：切换字符或课程步骤时停止旧音频和实时输入。
- **PS2-LEARN-005**：薄弱字符复习可以从首页、进度或字符详情发起，并预填内容包。

### 9.3 听抄与接收

- **PS2-RX-001**：听抄入口以内容预设命名，不以内部引擎名称命名。
- **PS2-RX-002**：字母、数字和混合随机组使用同一“随机字符组”预设，通过字符集筛选区分。
- **PS2-RX-003**：训练台至少配置 Character WPM、Effective WPM、音调、音量、内容长度和重播规则。
- **PS2-RX-004**：逐题模式可即时反馈；连续模式在结束或用户主动检查前不打断播放显示答案。
- **PS2-RX-005**：纸笔模式不要求用户逐题输入，但必须提供可控的答案对照和自评入口。
- **PS2-RX-006**：自定义文本在开始前完成字符合法性检查，不支持字符不得静默删除。
- **PS2-RX-007**：噪声和衰落默认关闭，不得在入门训练中自动启用。

### 9.4 发报与节奏

- **PS2-TX-001**：目标跟发和自由拍发共享输入、声音和安全释放链路。
- **PS2-TX-002**：按键按下开始发声，释放停止；失焦、后台和取消事件强制释放。
- **PS2-TX-003**：单键按时长判定点划；双键由按键类型决定符号。
- **PS2-TX-004**：按压期间毫秒数、滑块和点/划状态持续刷新，释放后保留最终值。
- **PS2-TX-005**：目标跟发在停顿阈值后自动判定并清空，无需用户手动退格或清空才能继续。
- **PS2-TX-006**：内容正确率与节奏得分分别记录；缺少节奏评分实现时不得用内容正确率冒充。
- **PS2-TX-007**：自由拍发自动字符提交等待允许在 3–10 个点单位间调整，首次默认 6 个单位。
- **PS2-TX-008**：无法解码的组合保持可见并标记未知，不静默丢弃。

### 9.5 查询与转换

- **PS2-TOOL-001**：工具页区分 Morse 转换、专业查询和中文电码，不把转换行为计入训练。
- **PS2-TOOL-002**：文本 ↔ Morse 转换使用与训练、播放相同的字符映射和规范化规则。
- **PS2-TOOL-003**：Q 简语、CW 缩略语、程序信号和数字简码分别分类并可交叉搜索。
- **PS2-TOOL-004**：CQ 不归入 Q 简语；程序信号必须标记连写发送方式。
- **PS2-TOOL-005**：数字简码结果显示所用方案和适用场景。
- **PS2-TOOL-006**：中文电码结果显示数据源和版本；一字多码返回候选而不是随机选择。
- **PS2-TOOL-007**：用户可以从可训练的查询项发起听抄或跟发会话。

### 9.6 内容包与训练预设

- **PS2-CONTENT-001**：内容包具有稳定 ID、版本、类别、语言/地区、来源和许可字段。
- **PS2-CONTENT-002**：同一内容包可被多个训练预设引用，但不得复制为互不一致的数据。
- **PS2-CONTENT-003**：开始会话时保存内容版本和生成后的题目快照。
- **PS2-CONTENT-004**：内容包更新不改变历史会话的答案解释。
- **PS2-CONTENT-005**：自定义内容默认只保存在本机；导出和同步必须由用户明确选择。

### 9.7 统一会话与评分

- **PS2-SES-001**：所有计分训练使用准备、出题、作答、反馈、完成、暂停和中断状态。
- **PS2-SES-002**：逐题提交立即持久化，连续听抄按安全检查点保存进度。
- **PS2-SES-003**：训练预设决定题面和作答控件，会话引擎不按页面复制状态机。
- **PS2-SES-004**：重播次数、反应时间、首次答案和纠正后答案分别记录。
- **PS2-SES-005**：错误反馈同时提供正确文本和可播放声音，不只用颜色。
- **PS2-SCORE-001**：字符和点划输入先规范化再精确匹配。
- **PS2-SCORE-002**：未启用超时时不产生超时错误；暂停时间不计入反应时间。
- **PS2-SCORE-003**：不同功能域可以有不同指标，但共享基础 attempt 和 session 数据协议。

### 9.8 进度、设置与隐私

- **PS2-PROG-001**：进度页按基础学习、听抄接收、发报内容和发报节奏展示表现。
- **PS2-PROG-002**：样本不足时明确标记，不生成误导性趋势或薄弱排名。
- **PS2-PROG-003**：从薄弱项可以进入适用的识别、听抄或跟发训练。
- **PS2-SET-001**：设置继续分为外观、音频、输入、训练、数据与隐私、关于与帮助。
- **PS2-SET-002**：预设局部设置与全局默认分开；只有明确操作才保存为默认。
- **PS2-PRIV-001**：自由拍发、转换和自定义正文默认不写入训练日志。
- **PS2-PRIV-002**：清空、覆盖导入和未来云同步属于高风险操作，必须明确确认。

### 9.9 迁移规则

- **PS2-MIG-001**：现有 session、attempt、character stat、课程和设置记录原地可读，不因页面改名重置。
- **PS2-MIG-002**：旧 mode ID 通过显式映射关联 V2 训练预设，不批量篡改历史答案。
- **PS2-MIG-003**：旧深链接在可行时重定向到新页面；包含有效 sessionId 的会话和结果链接必须继续工作。
- **PS2-MIG-004**：PWA 更新不清除 IndexedDB；数据升级失败保留旧数据并报告可恢复错误。
- **PS2-MIG-005**：结构重构发布前必须对四个旧模式、自由拍发、课程恢复、统计和导入导出执行回归测试。

## 10. Morse、速度与音频规则

- 点 1 单位、划 3 单位、元素间隔 1 单位、字符间隔 3 单位、单词间隔 7 单位。
- 点单位由 Character WPM 决定；Farnsworth 只扩展字符和单词之间的间隔。
- Character WPM 不得低于 Effective WPM。
- 连续播放使用音频时间轴，不能让界面渲染或普通定时器累积改变节奏。
- 速度调整只影响尚未调度的后续声音。
- 信道模拟作为播放后处理或信号层能力，不得改变题目标准答案和基础 Morse 时序数据。

## 11. 专业内容规则

### 11.1 Q 简语、缩略语与程序信号

- Q 简语条目至少包含代码、提问含义、陈述含义、类别和适用说明。
- CW 缩略语与普通文本组按正常字符间隔发送。
- 程序信号需要明确其连写 Morse 序列和显示方式。
- CQ 作为常用呼叫缩略语处理，不纳入 Q 简语表。

### 11.2 数字简码

- 每套数字简码映射有稳定方案 ID 和说明。
- 训练题显示当前方案，结果可同时展示简码和原数字。
- 首版可以提供比赛常用子集，但不得宣称为唯一全球标准。

### 11.3 中文电码

- 中文电码使用明确码表来源、版本和许可。
- 汉字与四位代码的映射和四位代码到数字 Morse 的转换分成两步。
- 简繁体转换不能替代码表查询；异体字和一字多码需返回候选。
- 未收录字保持原位并给出错误，不自动编造编码。

## 12. 状态与异常处理

| 状态 | 产品行为 |
| --- | --- |
| 音频尚未解锁 | 显示明确启用按钮，不自动播放失败 |
| 音频初始化失败 | 提供重试、静音浏览和诊断提示 |
| 应用进入后台 | 结束实时音符，暂停或安全保存会话 |
| 离线 | 保持已缓存课程、训练和工具可用，显示非阻塞标记 |
| 本地存储不可用 | 说明进度无法保存，允许临时体验 |
| 内容包缺失或版本不可用 | 不开始计分会话，提供修复或替代内容 |
| 自定义文本含非法字符 | 开始前集中标记并提供替换建议 |
| 中文电码多候选 | 展示候选和码表信息，等待用户选择 |
| 旧链接 | 重定向或恢复；无法迁移时提供明确返回入口 |
| 会话异常中断 | 下次启动提供继续或结束会话 |

## 13. 成功指标与验收场景

### 13.1 结构重构指标

- 用户可以正确预测字母听抄、自由拍发和 Morse 转换分别位于哪个功能域。
- 从首页到任一现有训练不超过三次操作。
- 四个旧训练模式的历史结果在新进度页可继续查看。
- 旧会话恢复、错题重练和结果深链接没有阻塞性回归。
- 移动端一级导航不超过五项。

### 13.2 场景 A：旧用户升级

给定用户已有课程进度和多个旧模式会话，当应用升级到 V2 结构时，用户能从首页继续未完成训练，并在进度页查看旧记录；数据不被重置。

### 13.3 场景 B：随机字符组听抄

给定用户希望练习字母与数字，当其进入听抄并选择“随机字符组”时，可以在同一设置中选择纯字母、纯数字或混合，而不需要寻找三个重复页面。

### 13.4 场景 C：目标跟发与自由拍发

给定用户进入发报功能域，可以清楚选择计分的目标跟发或不记录正文的自由拍发；两者使用一致的键位、声音和安全释放规则。

### 13.5 场景 D：专业查询进入训练

给定用户查询某个 Q 简语，当条目支持训练时，可以从详情一次操作进入对应听抄或跟发设置，且训练答案与查询解释来自同一内容版本。

### 13.6 场景 E：中文电码异常

给定输入包含未收录字或一字多码，系统显示具体问题、码表版本和候选，不生成看似有效但无法追溯的结果。

## 14. 风险与验证计划

| 风险 | 影响 | 首个验证动作 | 退出条件 |
| --- | --- | --- | --- |
| 四大入口仍让用户混淆 | 导航重构失去价值 | 用典型任务走查低保真信息架构 | 用户能正确找到听抄、发报和转换 |
| 旧模式迁移破坏历史 | 用户数据和信任受损 | 建立旧 ID 到新预设映射回归测试 | 会话、结果、错题和统计连续 |
| 内容包模型过度复杂 | 开发成本增加 | 先用字符和词组两个内容包验证 | 多域复用且不复制业务规则 |
| 连续听抄状态难恢复 | 长训练结果丢失 | 设计安全检查点与中断恢复实验 | 恢复时不重复或跳过已确认内容 |
| 专业术语或数据错误 | 训练错误知识 | 建立来源、版本和审校流程 | 查询和训练引用同一已审校数据 |
| 中文电码来源不清 | 法律和正确性风险 | 专项确认码表与许可 | 来源、版本、授权和异常策略明确 |

## 15. 发布与平台策略

1. 先在 Web/PWA 完成结构迁移和旧数据回归。
2. 冻结 V2 功能域、预设和内容包的稳定 ID。
3. 验证 Android/iOS Capacitor 容器的音频、触摸和深链接。
4. 验证 Tauri 桌面包装的键盘、窗口失焦和本地存储。
5. 专业内容按内容包独立迭代，不让内容更新改变核心训练协议。
